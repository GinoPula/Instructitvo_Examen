Primera parte · Pregunta 1 (1.5 pts)

Por qué separar entrenamiento y prueba

Sin cálculos aquí — es la base conceptual de todo lo que sigue.

Train / Test

Un modelo que se evalúa con los mismos datos que usó para aprender puede estar simplemente memorizando ruido, no aprendiendo la relación real. Separar una muestra de prueba nunca vista da una estimación honesta de cómo se comportará con datos futuros.

Data leakage

Información del test (o no disponible en el momento real de predecir) se filtra al entrenamiento. Ejemplos: estandarizar antes de partir la base, usar variables proxy del target, seleccionar variables con toda la base.

Este es el único punto del examen donde el enunciado permite y pide consultar bibliografía o IA — con prompt anotado y síntesis parafraseada, no copy-paste.

Para sustentar

Cuatro ideas que sostienen todo el examen

Si te preguntan algo transversal, probablemente sea una de estas cuatro.

1. Train / Test

Nunca evaluar con lo mismo que se usó para aprender — Pregunta 1, y de nuevo en Bloque 2 y Bloque 3 (80/20 con createDataPartition).

2. VP · FP · FN · VN

La misma base para toda métrica de clasificación del examen — a mano con 3 casos (KNN), con 18 (SVM), o con miles vía caret (Bloque 3).

3. exp(coeficiente)

El mismo truco de interpretación en las dos regresiones logísticas (Primera Parte y Bloque 3): pasa de log-odds a una razón de momios legible.

4. p-value vs. 0.05

Ya sea para un coeficiente individual, o para un bloque de variables comparando deviances con una chi-cuadrado (Bloque 2 y 3).
